Section

Sections are the building blocks of a template. Sections are fluid. They span 100% across the entire screen and are meant to have containers nested in them.

Video Backgrounds

Sections can optionally have a video background. The video wrapper needs to be a sibling above the container. If a section has a video background, the section needs to also have an overflow hidden rule to it.

The video source needs to be added to the "data-videos" attribute. In the unlikely event that different video's need to be loaded per viewport, the options are available.

Initializing Plugin

$('.section').drzSectionVideo();

Options

This plugin can take in multiple videos per viewport. Options are set using HTML attributes. You will need to pass them in JSON format. See above demo HTML. 

"desktop": {
  "src": "video url here", // url string
  "type": "mp4", // video type
  "inherit": false // should this video inherit from desktop (always false in desktop)
}
"tablet": {
  "src": "video url here", // url string
  "type": "mp4", // video type
  "inherit": true // should this video inherit from desktop
}
"mobile": {
  "src": "video url here", // url string
  "type": "mp4", // video type
  "inherit": true // should this video inherit from desktop
}

Overlay

Sections can optionally have an overlay layer which sits above the background color and image/video background layer. The overlay also needs to be a sibling above the container. If a section has an overlay, the section needs to also have an overflow hidden rule to it.
